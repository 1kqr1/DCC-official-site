# ブログ記事の書き方

このフォルダに Markdown ファイル（`.md`）を1つ追加するだけで、記事が1本増えます。
プログラムのコードを触る必要はありません。

## 手順

### 1. ファイルを作る

このフォルダ（`src/content/blog/`）に、新しいファイルを作ります。

ファイル名がそのまま記事のURLになります。**日付-英語の名前.md** の形がおすすめです。

```
2026-06-01-summer-lt.md
```

→ 記事URL: `/blog/2026-06-01-summer-lt`

### 2. 記事の中身を書く

ファイルの先頭に `---` で囲んだ「記事情報」を書き、その下に本文を書きます。

```markdown
---
title: 夏のLT会を開催しました
date: 2026-06-01
thumbnail: summer-lt.jpg
excerpt: 一覧に表示される短い紹介文をここに書きます。
tags: [イベント, LT会]
author: DCC広報
---

ここから下が本文です。

## 見出し

- 箇条書きも
- 使えます

**太字** や [リンク](https://example.com) もそのまま書けます。
```

| 項目 | 意味 | 必須 |
|------|------|------|
| `title` | 記事タイトル | ✅ |
| `date` | 公開日（`2026-06-01` の形式） | ✅ |
| `thumbnail` | サムネイル画像のファイル名（後述） | 任意 |
| `excerpt` | 一覧に出る紹介文 | 任意 |
| `tags` | タグ（`[ ]` の中にカンマ区切り） | 任意 |
| `author` | 書いた人 | 任意 |

### 3. サムネイル画像を置く

画像は `public/blog-images/` フォルダに入れます。
`thumbnail:` にはファイル名だけを書けばOKです（フォルダ名は不要）。

```
public/blog-images/summer-lt.jpg   ← ここに置く
thumbnail: summer-lt.jpg           ← こう書く
```

本文の中に画像を入れたいときも、同じフォルダに置いてファイル名だけで参照できます。

```markdown
![発表の様子](summer-lt-scene.jpg)
```

### 4. 反映する

ファイルを保存して GitHub に push すると、自動でサイトに反映されます。
記事は **日付の新しい順** に自動で並びます。

## よくある質問

- **記事を消したい** → その `.md` ファイルを削除するだけ。
- **下書きを置いておきたい** → ファイル名の先頭に `_` を付ける（例 `_draft-xxx.md`）と一覧に出ません。
  ※ このフォルダの `README.md` も `.md` ですが、`title` が無い記事は一覧に出ないよう作られています。
